# bug/prioridade/

Fila de priorização para bugs com maior impacto operacional.

## Objetivo
- Classificar urgência por risco funcional, impacto em usuário e custo de regressão.
- Determinar ordem de execução para `bug/fazer hotfix/`.

## Critérios de severidade sugeridos
- **P0**: quebra fluxo principal ou perda de dados.
- **P1**: degrada função crítica com workaround limitado.
- **P2**: problema relevante, sem bloqueio total.
- **P3**: ajuste incremental/documental.
