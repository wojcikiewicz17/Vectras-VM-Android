CORREÇÃO #1: ProcessSupervisor Race Condition - Sincronização de Failover
────────────────────────────────────────────────────────────────────────────
Arquivo: app/src/main/java/com/vectras/vm/core/ProcessSupervisor.java
Linhas Afetadas: 85-88, 24-25, 101-110
Tipo: Race Condition em ciclo de vida de processo

Problema Atual:
O campo volatile Process é acessado em métodos diferentes sem sincronização adequada.
Quando stopProcess() e killProcess() executam simultaneamente durante transição DEGRADED
para FAILOVER, ocorre acesso concorrente ao mesmo objeto Process causando double-free ou
null-pointer dereference. A volatilidade sozinha não garante segurança em operações multi-passo.

Solução Detalhada:
Implementar synchronized block em torno de todas as transições de estado do processo.
Criar método privado sincronizado chamado stopProcessSafely() que encapsula a lógica de
parada com garantia de que apenas uma thread executa destroy() ou destroyForcibly() por vez.

Código a Adicionar:
Após linha 84, adicionar novo campo para lock exclusivo:
private final Object processLock = new Object();

Após linha 110, adicionar novo método sincronizado:
private void stopProcessSafely(int timeoutSeconds) {
    synchronized (processLock) {
        if (process != null && process.isAlive()) {
            try {
                process.destroy();
                if (!process.waitFor(timeoutSeconds, TimeUnit.SECONDS)) {
                    process.destroyForcibly();
                    process.waitFor(3, TimeUnit.SECONDS);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
            process = null;
        }
    }
}

Modificar linhas 85-88 para chamar novo método:
Remover:
if (process != null && process.isAlive()) {
    process.destroy();
    if (!process.waitFor(3, TimeUnit.SECONDS)) {
        process.destroyForcibly();
        process.waitFor(2, TimeUnit.SECONDS);
    }
}

Substituir por:
stopProcessSafely(3);

Impacto: Elimina race condition, garante teardown ordentado, reduz ANRs em failover.

package com.vectras.vm.receiver;

import android.content.BroadcastReceiver;
import android.content.Context;
import android.content.Intent;
import android.net.Uri;
import java.io.InputStream;
import java.security.MessageDigest;

public class RomImportReceiver extends BroadcastReceiver {
    private static final String EXPECTED_ROM_HASH = "sha256hashvalue..."; // Definido em BuildConfig

    @Override
    public void onReceive(Context context, Intent intent) {
        Uri romUri = intent.getData();
        if (romUri == null) return;

        try {
            InputStream is = context.getContentResolver().openInputStream(romUri);
            String romHash = computeSHA256(is);

            if (!EXPECTED_ROM_HASH.equals(romHash)) {
                // Rejeitar ROM com hash inválido
                return;
            }

            // Proceder com importação apenas se hash válido
            Intent importIntent = new Intent(context, RomInfo.class);
            importIntent.setData(romUri);
            context.startActivity(importIntent);
        } catch (Exception e) {
            // Log erro de segurança
        }
    }

    private String computeSHA256(InputStream is) throws Exception {
        MessageDigest digest = MessageDigest.getInstance("SHA-256");
        byte[] buffer = new byte[8192];
        int count;
        while ((count = is.read(buffer)) > 0) {
            digest.update(buffer, 0, count);
        }
        byte[] hash = digest.digest();
        StringBuilder sb = new StringBuilder();
        for (byte b : hash) {
            sb.append(String.format("%02x", b));
        }
        return sb.toString();
    }
}

Impacto: Elimina vetor de injeção, restringe importação a apps assinados apenas.

private String[] parseCommandLine(String cmdline) {
    // Usa algoritmo de parsing que respeita espaço/quotes sem interpretar shell operators
    java.util.List<String> parts = new java.util.ArrayList<>();
    StringBuilder current = new StringBuilder();
    boolean inQuotes = false;

    for (int i = 0; i < cmdline.length(); i++) {
        char c = cmdline.charAt(i);

        if (c == '"' || c == '\'') {
            inQuotes = !inQuotes;
        } else if (c == ' ' && !inQuotes) {
            if (current.length() > 0) {
                parts.add(current.toString());
                current = new StringBuilder();
            }
        } else {
            current.append(c);
        }
    }

    if (current.length() > 0) {
        parts.add(current.toString());
    }

    return parts.toArray(new String[0]);
}

CORREÇÃO #5: NativeFastPath.nativeCopyBytes() Integer Overflow - Bounds Check
────────────────────────────────────────────────────────────────────────────
Arquivo: app/src/main/cpp/vectra_core_accel.c
Linha Afetada: 85
Tipo: Buffer overflow via integer overflow em aritmética de ponteiro

Problema Atual:
Pointers são calculados como (uint8_t*)d + dstOffset e (uint8_t*)s + srcOffset sem verificação
de overflow. Se dstOffset + length > array_length, escrita ocorre fora do heap causando
memory corruption. Verificações em linha 77 apenas checam se valores são negativos, não se
a soma ultrapassa bounds.

Solução Detalhada:
Adicionar verificação de bounds ANTES de qualquer aritmética de ponteiro.
Modificar função a partir de linha 75 para:

JNIEXPORT jint JNICALL
Java_com_vectras_vm_core_NativeFastPath_nativeCopyBytes(JNIEnv* env, jclass clazz,
    jbyteArray src, jint srcOffset, jbyteArray dst, jint dstOffset, jint length) {
    (void)clazz;

    // Validação inicial
    if (vectra_kernel_ensure() != RMR_UK_OK || !src || !dst) return -1;
    if (srcOffset < 0 || dstOffset < 0 || length < 0) return -2;

    // Obter comprimentos reais dos arrays
    jsize srcLen = (*env)->GetArrayLength(env, src);
    jsize dstLen = (*env)->GetArrayLength(env, dst);

    // Verificar overflow: srcOffset + length não deve exceder srcLen
    if (srcOffset > srcLen || (srcLen - srcOffset) < length) return -3;

    // Verificar overflow: dstOffset + length não deve exceder dstLen
    if (dstOffset > dstLen || (dstLen - dstOffset) < length) return -4;

    // Agora é seguro acessar arrays
    jbyte* s = (*env)->GetPrimitiveArrayCritical(env, src, NULL);
    jbyte* d = (*env)->GetPrimitiveArrayCritical(env, dst, NULL);

    if (!s || !d) {
        if (s) (*env)->ReleasePrimitiveArrayCritical(env, src, s, JNI_ABORT);
        if (d) (*env)->ReleasePrimitiveArrayCritical(env, dst, d, JNI_ABORT);
        return -5;
    }

    // Cópia segura
    int rc = RmR_UnifiedKernel_Copy(&g_unified_state,
        (uint8_t*)d + dstOffset,
        (const uint8_t*)s + srcOffset,
        (size_t)length);

    (*env)->ReleasePrimitiveArrayCritical(env, src, s, JNI_ABORT);
    (*env)->ReleasePrimitiveArrayCritical(env, dst, d, 0);

    return (jint)rc;
}

Impacto: Elimina exploração de buffer overflow, garante acesso dentro de bounds.

private boolean validateCommandArray(String[] cmd) {
    if (cmd == null || cmd.length == 0) return false;

    for (int i = 0; i < cmd.length; i++) {
        String part = cmd[i];
        if (part == null || part.isEmpty()) {
            Log.e(TAG, "Command array element " + i + " is null or empty");
            return false;
        }

        // Verificar caracteres controle e null bytes
        if (containsControlCharacters(part)) {
            Log.e(TAG, "Command array element " + i + " contains control characters");
            return false;
        }
    }

    return true;
}

private boolean validateEnvironmentArray(String[] env) {
    if (env == null) return true; // null é válido (herda parent)

    for (int i = 0; i < env.length; i++) {
        String entry = env[i];
        if (entry == null || !entry.contains("=")) {
            Log.e(TAG, "Environment entry " + i + " is not valid KEY=VALUE format");
            return false;
        }

        String[] parts = entry.split("=", 2);
        if (parts[0].isEmpty() || parts.length != 2) {
            Log.e(TAG, "Environment entry " + i + " has invalid format");
            return false;
        }

        if (containsControlCharacters(entry)) {
            Log.e(TAG, "Environment entry " + i + " contains control characters");
            return false;
        }
    }

    return true;
}

private boolean containsControlCharacters(String s) {
    for (int i = 0; i < s.length(); i++) {
        char c = s.charAt(i);
        if (c < 32 || (c >= 127 && c <= 159)) {
            return true; // Encontrou controle/null character
        }
    }
    return false;
}

Impacto: Previne buffer overflow em libc, falha early com erro claro.

Criar versão com error handling:
private static final QmpTransport DEFAULT_QMP_TRANSPORT = () -> {
    try {
        String response = QmpClient.sendCommandForStopPath("{ \"execute\": \"system_powerdown\" }");
        if (response == null || response.contains("error")) {
            Log.w(TAG, "QMP powerdown failed or returned error: " + response);
            throw new IOException("QMP command failed");
        }
        return response;
    } catch (IOException | TimeoutException e) {
        Log.e(TAG, "QMP communication failed, will fallback to process.destroy()", e);
        throw new RuntimeException("QMP communication failed, triggering fallback", e);
    }
};

Modificar sequência de parada em ProcessSupervisor.stopProcess():
private void stopProcessSafely(long timeoutMs) {
    synchronized (processLock) {
        if (process == null) return;

        try {
            // Tentar via QMP primeiro
            try {
                qmpTransport.sendPowerdown();
                Log.i(TAG, "QMP powerdown sent successfully");

                // Aguardar processo terminar via QMP (graceful)
                if (process.waitFor(5, TimeUnit.SECONDS)) {
                    Log.i(TAG, "Process terminated via QMP");
                    return;
                }
            } catch (Exception qmpError) {
                Log.w(TAG, "QMP method failed, falling back to process.destroy()", qmpError);
                // Continuar para destroy() fallback abaixo
            }

            // Fallback: destroy() se QMP falhou
            process.destroy();
            if (!process.waitFor(3, TimeUnit.SECONDS)) {
                Log.w(TAG, "Process not terminated by destroy(), forcing destroyForcibly()");
                process.destroyForcibly();
                process.waitFor(2, TimeUnit.SECONDS);
            }

        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            Log.e(TAG, "Interrupted while stopping process", e);

        }
    }
}
