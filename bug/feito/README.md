# bug/feito/

Arquivo de histórico para hotfix concluído e validado.

## Quando mover para `feito/`
- Critério de aceite da issue atendido.
- Evidência de validação registrada (teste, log ou reprodução).
- Referência cruzada para commit/patch disponível.

## Convenção
- Manter nome do artefato original sempre que possível.
- Incluir resumo de fechamento no topo do arquivo movido.
